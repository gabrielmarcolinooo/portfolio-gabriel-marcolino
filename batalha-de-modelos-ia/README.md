# 🤖 Batalha de Modelos & Engenharia de Prompt (XML)

## 📝 Descrição do Projeto
Este projeto consiste em uma análise comparativa entre diferentes modelos de Inteligência Artificial utilizando técnicas de **Engenharia de Prompt** com estrutura em XML. O objetivo principal foi avaliar como diferentes IAs interpretam e executam um mesmo prompt estruturado para geração de uma página HTML Single Page com CSS integrado.

Desenvolvido como parte da disciplina de **Engenharia de Prompt e Aplicações em IA**, o projeto envolveu testes realizados em diversas ferramentas de IA, incluindo ChatGPT, Gemini, Claude, Qwen, DeepSeek, Grok e Maritaca. Durante a atividade, foram analisados aspectos como precisão do HTML gerado, criatividade do conteúdo, quantidade de bugs, qualidade visual e quantidade de tokens utilizados por cada modelo.

## 🚀 Tecnologias Utilizadas
* **Área:** Inteligência Artificial Generativa
* **Técnicas:** Engenharia de Prompt em XML
* **Linguagens:** HTML5 e CSS3
* **Ferramentas Testadas:** ChatGPT, Gemini, Claude, Qwen, DeepSeek, Grok e Maritaca
* **Conceitos:** Prompt Engineering, comparação de modelos e geração automática de código

## 📊 Resultados e Aprendizados
O projeto demonstrou diferenças significativas entre os modelos de IA durante a geração do código HTML.

* **Claude:** Apresentou os melhores resultados em estrutura, design e ausência de bugs.
* **ChatGPT e Gemini:** Obtiveram bons resultados, com páginas funcionais e organizadas.
* **Qwen e Maritaca:** Demonstraram maior quantidade de erros e baixa fidelidade ao prompt.
* **Análise de Tokens:** Foi possível comparar o nível de detalhamento das respostas geradas por cada IA.

Além disso, o projeto permitiu compreender:
* Como prompts estruturados em XML influenciam a saída dos modelos.
* Diferenças de interpretação entre IAs generativas.
* A importância da clareza e especificidade na Engenharia de Prompt.
* O impacto dos tokens na qualidade do resultado final.

## 🔧 Como Executar
1. Escolha uma IA generativa compatível.
2. Utilize o prompt estruturado em XML desenvolvido no projeto.
3. Gere a página HTML proposta.
4. Compare os resultados entre diferentes modelos de IA.
5. Analise estrutura, design, bugs e quantidade de tokens utilizados.
